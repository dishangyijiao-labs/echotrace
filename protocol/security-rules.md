# 安全规则

每次编码前读取本文件。安全是每个阶段的前置条件，不是事后补救。

---

## 认证与密钥

- **不要在代码中硬编码密钥或默认密码。** 缺少必需密钥时，应用必须启动失败。
- **不要提交 `.env` 文件、API 密钥、token 或凭据。** 确认 `.gitignore` 覆盖了 `.env`、`*.pem`、`*.key`。
- **密码必须加密存储。** 不得存储、记录或返回明文密码。

## 输入验证与注入防护

- **不要用字符串拼接构建 SQL。** 只使用参数化查询。
- **在 API 边界验证和清理所有用户输入。** 不要将原始用户输入传递给 `eval()`、`exec()` 或 shell 命令。
- **文件上传必须验证文件类型和大小。**

## API 安全

- **所有写入端点必须有速率限制。**
- **CORS 来源必须显式指定。** 不要使用 `allow_origins=["*"]`。
- **错误响应不得泄露内部细节。** 生产环境返回通用错误消息，堆栈跟踪只发到监控系统。

## 依赖安全

- **锁定依赖版本。** 定期检查已知漏洞。
- **添加新依赖前先审查。** 检查：维护状态、已知 CVE、许可证兼容性。

## 编码前检查清单

- [ ] 用户输入是否经过验证和清理
- [ ] 错误信息是否泄露内部细节
- [ ] 新增的 API 端点是否有鉴权和速率限制

## 提交前检查清单

- [ ] 是否有硬编码的密钥或凭据
- [ ] .gitignore 是否覆盖敏感文件
- [ ] 新增的依赖是否有已知漏洞
